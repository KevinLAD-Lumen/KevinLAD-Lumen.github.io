严重级别（Severity Level）

LOG可以指明如下几个严重级别（升序）
INFO，WARNING，ERROR，FATAL
其中，当用户程序记录一个FATAL信息之后，程序会被终止。
注意： 指定严重级的信息不只会记录在相应级别的log中，还会记录在所有低严重级的log中。
例如，当程序记录一个FATAL信息的时候，相应信息会分别出现在INFO，WARNING，ERROR，FATAL级别的日志中。

此外，在Debug模式下（如没有define NDEBUG宏），DFATAL级别可以记录FATAL级别的错误但会通过自动降低严重级别到ERROR来避免挂起程序。

除非特殊指明路径，glog会默认记录在
"/tmp/<program name>.<hostname>.<user name>.log.<severity level>.<date>.<time>.<pid>"
例如：
"/tmp/hello_world.example.com.hamaji.log.INFO.20080709-222411.10474"

默认情况，glog也会复制ERROR和FATAL的日志信息到stderr。

RRTS中可以通过引入log.h， 来使用LOG_INFO,LOG_ERROR等简写宏定义。

条件记录日志

glog可以控制日志信息在指定条件下进行记录。具体使用如下：

  1. LOG_IF(severity, condition)

上面的语句表示，只有当num_cookies > 10条件成立时，“Got lots of cookies”日志信息才被记录。

  1. LOG_EVERY_N(severity, n)

上面的语句表示，在程序中周期性的记录日志信息，在该语句第1、11、21……次被执行的时候，记录日志信息。COUNTER变量表示该语句被执行的次数。

  1. LOG_IF_EVERY_N(severity, condition, n)

上面的语句为1，2项功能的合并，size>1024的条件连续成立10次的时候记录日志信息。COUNTER变量表示该条件成立的次数。

  1. LOG_FIRST_N(severity, n)

上面的语句表示，当该语句只在首次执行了20次以后记录日志信息， COUNTER变量表示该语句被执行的次数。

RRTS中可以通过引入log.h， 来使用LOG_INFO_EVERY_N等简写宏定义。

CHECK宏

为了及早发现程序错误，glog提供了CHECK宏用来在调试中有条件的终止程序。当不满足指定条件时，CHECK宏会终止程序，并记录相应日志信息。类似于c标准库中的assert，但CHECK宏也可以用于release版程序（不受NDEBUG限制）。

由CHECK宏派生出许多简便实用的宏，下面举一些常见的例子。

1. 大小关系检测

Macros Description
CHECK_EQ(var1,var2) if var1 == var2, then pass
CHECK_NE(var1,var2) if var1 != var2, then pass
CHECK_LE(var1,var2) if var1 <= var2, then pass
CHECK_LT(var1,var2) if var1 < var2, then pass
CHECK_GE(var1,var2) if var1 >= var2, then pass
CHECK_GT(var1,var2) if var1 > var2, then pass

这些宏会检测两个参数的大小关系，不符合预期条件时会记录FATAL日志。检测类型不一致时，可以使用static_cast转换类型。相应信息通过流来传递。
