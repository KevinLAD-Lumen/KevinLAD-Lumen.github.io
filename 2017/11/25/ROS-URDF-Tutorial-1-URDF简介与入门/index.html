URDF教程 (1): URDF简介与入门

2017-11-25
ros

在ROS中，当我们要对机器人进行仿真模拟时，便需要对机器人进行建模。而URDF ( Unified Robot Description Format,意为统一机器人描述格式）正是一种用来代表机器人模型的XML代码格式。本文讲简单的对URDF进行介绍，并尝试建立一个差速轮小车的模型。教程最后会给出一系列的参考网址。在之后的教程中，我们会使用Xacro来简化URDF文件，还会添加Gazebo的相关内容，来实现Gazebo的仿真。

运行环境：

  • Ubuntu 16.04
  • ROS Kinetic
  • Gazebo7

首先，我们先来安装URDF_Tutorial包。我们可以利用里面的launch文件来进行下面的步骤。

sudo apt-get install ros-kinetic-urdf-tutorial

URDF以一种树状结构来描述一个机器人，树结构主要由两个元素构成：link与joint。在URDF中，link元素以一个刚体的物理性质（尺寸，初始位置，颜色等等）来描述该刚体。Link元素之间由joint元素相连接。Joint元素描述了刚体间的的运动学与动力学关系。（刚体间的连接，关节的类型，旋转轴，阻尼，摩擦，等等）

上面XML中，我们首先定义了一个Robot，起名叫dd_robot，同时目前他只包含一个link，我们管他叫base_link，以它来代表我们的小车车身。从外观上看，我们把它定义为了一个0.5米长，0.5米宽，0.25米高的方块盒子，并且初始位置为环境的最中心（0,0,0），并且在roll，pitch，yaw轴都没有旋转。在URDF模型中，通常要以base_link作为root link，以此为支出来搭建出整个机器人的运动学模型。

下一步，我们要给我们的小车装上轮子。每当我们要添加一个link元素到URDF模型中的时候，我们都必须添加joint元素来形容link之间的关系。Joint元素定义了这个关节是可以运动的或者固定不动的。对于可以移动的关节，URDF描述了关节的运动学与动力学性质以及相应的安全极限。在URDF中，有以下六种可能的joint类型。

  • Fixed:这种joint其实并不是一个真正意义上的“关节”，因为它并不能移动。它的每一个自由度都是上了锁一样的固定住的。这种关节不需要轴、校准、动力学、限位以及可靠的控制器。

  • Revolute：此种关节可以绕着一个轴旋转，并且有特定指明的上限和下限。

  • Continuous：此种关节代表了连续的铰链接（continuous hinge joint），可以绕着一个轴来连续的旋转，没有上限与下限。

  • Prismatic：这是一种滑动连接，（类似小滑块）,可以沿着某个轴滑动，并且有特定的滑动上限下限范围。

  • Floating：此种连接方式在所有的自由度上都是可以自由移动的(六自由度万向节)。

  • Planar：这种连接方式可以自由的在垂直于轴线的平面中运动。

我们在修改后的URDF文件中做了以下几件事：

  • 每个轮子定义为0.2m半径，0.1米长的圆柱。每个轮子的圆心都在车身的中心轴上，并且每个轮子绕x轴旋转了1.560795（= pi/2=90 degrees）。

  • 每个joint依照着一个父节点和一个子加点来定义。base_link是我们模型中的root link，我们的轮子定义的位置都是按照base_link的相对位置来描述。

  • 轮子的joint由父参考系来定义。所以轮子节点的位置是x轴的+0.3m和-0.3m。

  • 这些joint元素完整的定义了整个机器人的运动学模型。
